# Security Policy

## Supported versions

This project is currently maintained in its latest development state.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.
Instead, report them privately to the maintainers so they can be addressed responsibly.

## Disclosure process

1. Report the issue privately.
2. A maintainer will acknowledge receipt.
3. A fix will be prepared and released as appropriate.
4. Public disclosure will follow once a safe resolution is available.
