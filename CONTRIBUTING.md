# Contributing

Thank you for your interest in contributing to API Monitoring Starter.

## How to contribute

1. Fork the repository.
2. Create a feature branch from main.
3. Make your changes and add tests when relevant.
4. Run the relevant Maven tests.
5. Open a pull request with a clear summary.

## Development setup

```bash
git clone <your-fork-url>
cd API-Monitoring-Starter
./mvnw test
```

## Coding guidelines

- Keep changes focused and easy to review.
- Prefer clear naming and small, well-documented commits.
- Update or add tests for behavior changes.
- Avoid introducing unnecessary dependencies.

## Pull request checklist

- [ ] Tests pass locally
- [ ] Documentation is updated when needed
- [ ] The change is scoped and clearly described
- [ ] No unrelated files are included
